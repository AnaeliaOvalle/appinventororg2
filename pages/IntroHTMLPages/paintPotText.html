Changing the Color

If we want to change the color of lines being drawn, which component are we talking about? (Hint: Which component does the drawing of lines.)


Look through the Canvas Component Property blocks (green blocks) and find the Set Canvas1.paintcolor. This block will change the color that is painted with.

Place this block (and its corresponding color) inside the event handler for each color button. Colors can be found in the colors drawer in the blocks editor on the left hand side.

The Logic

When the Green button is clicked, set the Canvas’ PaintColor to Green.
Remember: Colors are found in the blocks editor’s built-in blocks under “Color.”


Test that the color buttons change the color that is drawn.

Custom Canvas Image

Taking a picture and putting it as the Canvas Background Image is pretty simple. First make sure that you have the Camera Component in your app.

Then you want to think, “When do I want to allow the camera component to turn on to allow the user to take a photo?


The answer to this question is probably when the TakePicture Button is pressed. So for this button’s event handler (TakePicture_Button) you’re going to put in take picture.

Once the picture has been taken and saved, the Camera Component’s AfterPicture event handler goes off. What do we want to do after the picture is taken? Set Canvas.BackgroundImage to the Image that was taken by the camera.

Drawing Different Size Lines

Part of the task is to allow the user to be able to change the line size that they are drawing with by having the user interact with something to make the lines bigger or smaller.


So you are going to need a visual component that will allow the user to control line size. This can be a textbox, slider, + and - buttons, or whatever you want.

Changing line size is fairly easy because the canvas component has a property for line size that can be changed anywhere.

You want the line width to change to what the user wants. You can use a Button and a Texbox, a slider, or even plus or minus buttons with a variable.

Changing the Color: Know Which Color is Pressed

Giving the user feedback to let them know that their tap on the screen was successful or the current settings are important in any app. In the example below, the user knows which menu option they are in because the bottom menu icon is highlighted telling the user that they are currently in the Keypad menu.

In this app, we want the user to know what color they are about to draw with before they draw. This can happen a number of ways.

Options:

  • You could make the ColorButton’s text that is being pressed bold
  • You can make the background color change to the color selected
  • You could add in a button with no text and have that button’s color change to the current color each time the user presses on a color button

In the snapchat example below, the color that will be drawn with is turquoise and one knows that because it is the color of the box on the top left corner).

Logic Hint

If you are trying to identify the current color that will be drawn with, you have to identify the colors that won’t be drawn. That means if you change the clicked button’s text to bold, tthe other buttons should no longer be bold.

For example if you have 3 colors, each time you hit Red you want the paint color to change to red, the RedButton.fontbold to true and you want the blackbutton.fontbold to false and the whitebutton. fontbold to false. This way only the red button is bold while the rest are not bold.
