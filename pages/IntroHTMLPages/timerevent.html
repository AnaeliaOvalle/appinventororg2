The Clock Component Timer Event: How does it work?

The Clock.Timer event below is like an alarm clock, it will do the event at the end of the timer. Just like how a timer, on a phone, will ring when it gets to 0, instead of ring,we use this moment as an event handler within the app and this timer can be set on a loop (to repeat) or it can be set to go off once.
In the example below, the Clock.timerInterval property, determines how long it will take for the sound to play. By default it is set to 1000 milliseconds (1 second), so if you donâ€™t change the timer interval, the function call "Sound1.Play" will happen every second.
You can set, or change, this property in the designer or with blocks and you can decide whether you want the timer to repeat or stop after it goes off (TimerAlwaysFires).
