The Clock Component Timer Event: How does it work?

The Clock.Timer event is like an alarm clock, it will do the event at the end of a timer. Just like how a timer will ring when it gets to 0, instead of ring, we can tell the clock component to do anything within our app and it can be set to loop (repeat) or go off once.
The Clock.TimerInterval property, determines how long it will take for the sound to play. By default it is set to 1000 milliseconds (1 second), so if you donâ€™t change the timer interval the sound above will play every second.
You can set this property in the designer or with blocks and you can decide whether you want the timer to repeat or stop after it goes off.
