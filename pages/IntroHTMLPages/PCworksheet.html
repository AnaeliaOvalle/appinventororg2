Properties

Component properties are the attributes of a component-- they define how the component will appear (and other things).

For instance, the Button.Text property defines the text that appears on a button. You can set the value of a property in the designer, which changes the initial value. But you can also change a property as an app runs with blocks, using a set block:

Conditionals

Conditionals allow the app to ask questions and determine which blocks to execute based on the answer. Consider again the conditional (if-else) blocks used in the “I Have a Dream” app:

When a button is clicked, If a speech is already playing, it will pause it. Otherwise (else) it will start it. The blocks in the “then” slot and “else” slot are only conditionally executed.

Practice

To practice using If-conditionals and properties, sketch blocks for:

  1. Each time a button named “Button1” is clicked, its text toggles between “Start” and “Stop”. So the first time it is clicked, the text on the button changes from “Start” to “Stop”. The second time it changes back to “Start”, and so on.
  2. Each time the button is clicked, it toggles between three values, “A”, “B”, and C”.
  3. Each time the button is clicked, it shows one of three different images (e.g., pics of MLK, Malcolm X, and Rosa Parks).
